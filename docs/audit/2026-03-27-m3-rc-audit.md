**Date**: 2026-03-27  
**Scope**: Functional closure for last large pre-release (F-19, F-20, F-21, F-22, F-24), export facade, new dependencies.

## Verification

- `npm run build` — pass  
- `npm run lint` — pass  

## Security

1. **XSS / HTML injection**  
   - Preview and export still render **user-controlled** project HTML via `iframe` + `document.write` (full document built from `renderProject`). This is intentional for a WYSIWYG exporter.  
   - **Theme stylesheet URLs** are restricted to `https://` and validated with `URL` before emitting `<link href="…">`. No `javascript:` or `http:` on an HTTPS app.  
   - **Liquid export** wraps HTML in `{% raw %}…{% endraw %}`. Content containing `{% endraw %}` can break Liquid; documented in UI.

2. **Secrets**  
   - AI keys remain in existing client-side storage; **not** logged. OCR runs **locally** (Tesseract); images are not sent to a third party by this path.

3. **Dependencies**  
   - **jszip** (MIT) — used only for client-side ZIP generation.  
   - **tesseract.js** (Apache-2.0) — dynamically imported; increases lazy-loaded chunk size; English worker loaded on first OCR use.  
   - `npm audit` may report upstream advisories; no `npm audit fix --force` applied in this release.

## Type safety & data

- `parseProjectJson` accepts new optional fields: `includeSrcsetInExport`, `previewThemeStylesheetUrls`, `includeThemeStylesheetUrlsInExport`.  
- `updateProject` pick-list extended for the same fields.

## Residual risk / follow-ups

- Lighthouse and automated E2E still manual (see `memory-bank/progress.md`).  
- Theme CSS may fail to apply in the iframe due to **CORS** or **MIME** rules on third-party CDNs — documented for operators.  
- Full Liquid parameterization per slot is **out of scope** for this RC.
